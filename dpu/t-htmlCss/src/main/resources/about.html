DPU takes HTML files as inputs and for each file generate a tree structure of statements based. The root a of a tree is called ?rootSubject and is unique for each input file. Besides that every tree is located in it's own output graph (entry). Tree structure is determined by used provided options.
Trees are generated by recursive application of actions on data objects (entities).
To given an example lets have following html page:
<html> <body>
<div>
<a href="link.html">My link</a>
</div>
</body> </html>
Our task is to extract the value "link.html" from given html and store it as a object in output RDF statement.

The input entity is called webPage. This entity represent the whole page.

In first step we use action QUERY with Action data set to "a" (see table bellow for list of all actions). This action takes input (webPage), execute given JSoup query (Action data) and put result to output entity.

Let's call output entity Link. Now the content of Link is a single element <a href="link.html">My link</a>

As a next step we extract a link (value of href attribute) from the Link element, for this we utilize action called ATTRIBUTE with Action data set to "href".

Again action do the transformation of given input entity to output, but this time output entity is a text - not an element as before. The text is equal to "link.html". The difference between entity types (see table bellow for list of all used entity types) is important as we can not use for example QUERY (JSoup query) with text entity as input (see table bellow for list of action input/output types).

Now we can output the text as a object in RDF statement. To to this we use action OUTPUT. Action data of this action determine used predicate, in our case we can use value "http://localhost/ontology/temp/link". This action has no output, instead it created a statement.

As an object value of input text entity is used, as a predicate user Action data value is used. But what about subject? Well each entity carry a URI of subject as a property and this is where the subject is taken from. In our case as we haven't set this property to any value so default subject is used.
List currently supported types:
  • element - a single HTML element, like <a> or <div>
  • list of elements - list of multiple elements
  • text - a text

List currently supported properties:
  • subject - carried by each entity. A text representation of a subject URI.

Command overview:
Command action data input entity
type		 output entity
type		 Description
QUERY JSoup query [list of]element [list of]element Execute JSoup query. The output type is determined by the query result.
TEXT none an element a text Select content as string.
HTML none an element a text Select content as html string.
ATTRIBUTE name of the attribute an element a text Select attribute of given name.
UNLIST none list of elements element Given list of elements put each element into separated group.
SUBJECT_CLASS URI - class [list of]element [list of]element Set given URI as a class for a current subject.
OUTPUT URI - predicate a text - Create statement with input entity as an object, given value as a predicate and last defined subject.
SUBJECT [list of]element [list of]element [optional]
URI - predicate		 Create new subject entity and store to given entities as a property. The value of subject is determined by DPU and can not be customized.
Action data are used to determine predicate used to join new subject to old subject (?oldSub ?predicate ?newSub). If no predicate is given then default predicate is used (can be set on 'Advanced settings tab).

Advanced settings
  • 'Root subject class' - class of root subject.
  • 'Default has predicate' - used as a default Action data data for all SUBJECT actions
  • 'Generate source info' - if checked then statement is generated: ?rootSubject <http://unifiedviews.eu/ontology/e-htmlCss/source> ?inputFileSymbolicName.
